Design notes — Pattern B: in-surface inline error alert (design/spike/error-handling.md)

  • Anatomy — one shared InlineErrorAlert: tinted abort-channel panel (bg-error/10 + border-error/30 + rounded-[10px]), an fx-led dot in text-error top-aligned beside the message (13px default; compact variant 12px for the chat strip, the sign-out confirm row and the dump composer). Never a solid filled red block — this replaces the daisyUI alert alert-error everywhere.
  • Placement rule — the alert renders inside the surface, directly above its action row: task-modal footer, plan-form sticky summary bar, review-modal footer, AI-chat footer strip, dump composer, sign-out confirm row, login's Google button. The submit/confirm button next to it is the retry — no separate retry affordance.
  • Lifecycle — no auto-dismiss (the user is mid-task); the error clears on the next attempt (setError(null) at submit start) and isSubmitting resets on failure so Cancel/submit re-enable. Every screen here is pinned post-failure with buttons live again.
  • What each screen replaces — task modal, plan form, review confirm and dump swap the FormErrorAlert call sites (TaskModalFooter, PlanForm, DumpComposer; ReviewChangesPanel gains a slot it lacks today). The three AI screens replace the hand-rolled alert-error strip in AiChatModalContent — generate and approve share the one footer slot above the input. Settings sign-out and login are new surfaces: both paths are fully silent today.
  • AI init failure — a real in-body state (error tile + message + primary Try again re-firing init) replaces today's infinite LoadingBubble hang; the input bar stays disabled until init succeeds.
  • Sign-out & login semantics — sign-out failure keeps the user in the settings sheet (today it navigates to /auth/login regardless); the login alert serves both the OAuth call failure and the auth callback's new ?error=1 redirect (today the callback redirects with no signal and the button gives zero feedback).
  • Plan-form domain error — "An active plan already exists." is user-actionable copy, kept specific on purpose; today it is hardcoded English inside planService, here it becomes Errors.activePlanExists so no string crosses the wire.
  • i18n keys per screen — task modal / review confirm / dump → Errors.saveFailed (new); plan form → Errors.activePlanExists (new); AI generate → Errors.generateFailed (existing); AI approve → Errors.createFailed (existing); AI init → Errors.aiInitFailed (new); sign-out → Errors.signOutFailed (new); login → Errors.signInFailed (new). All generic, sentence case, no internals ever.
