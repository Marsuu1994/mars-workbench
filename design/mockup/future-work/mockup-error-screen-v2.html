Design notes — Pattern C page error screen (see design/spike/error-handling.md)

  • File map: error.tsx under kanban/, kanban/priorities/, kanban/dump/, and kanban/plans/, plus root global-error.tsx and not-found.tsx — all six render one shared ErrorScreen domain component (domain/shared/), parameterized by voice (error / info), actions, and micro-label.
  • Skeleton → error swap: loading.tsx shows the pulsing skeleton while the server renders; if the render throws, Next swaps the same segment slot to error.tsx — the sidebar/dock survive because the root layout still renders. Try again is wired to error.tsx's reset(), which re-renders the segment (skeleton shows again); the ghost link navigates to /kanban.
  • Why sync failures land here: syncService.ensureSynced runs during the GET render of every kanban page, so a failed sync write takes down the whole page render — the page error screen is also the sync-failure surface. Its writes are transactional, hence the body copy's "your data is safe". Copy stays generic; the real error is logged server-side.
  • Segment layouts survive too: the plans segment's error.tsx renders inside PlanChrome (header + max-w-xl body column) exactly like its loading.tsx does — the error state never breaks planning mode's frame.
  • Channel discipline: the abort channel colors the state — icon tile border-error/30 bg-error/10 text-error, "SIGNAL LOST" micro-label (fx-label) with a pulsing fx-led — while the action button stays primary (action channel), same rule as patterns A and B. Never a filled red block.
  • global-error.tsx: when the root layout itself crashes there is no chrome to survive — the variant is self-contained (renders its own <html>/<body>): brand mark (LoginScreen's glowing tile + wordmark) top-center over the bare console-sky background, then the same composition.
  • 404 voice: not-found.tsx (used by the edit-plan notFound() today) keeps the composition but swaps to the info/neutral voice — compass tile and "404 — NOT FOUND" micro-label in info, static LED, single primary action "Back to board". A missing page is not an abort.
  • Copy / i18n: proposed Errors keys — screenTitle ("Something went wrong"), screenBody ("We couldn't load this page — your data is safe."), tryAgain, backToBoard, notFoundTitle ("Page not found"), notFoundBody ("This page doesn't exist or may have been moved."). The mono micro-labels ("SIGNAL LOST", "404 — NOT FOUND") are chrome telemetry and join the same namespace as screenLabel/notFoundLabel.
