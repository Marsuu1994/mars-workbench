Mobile Backlog — Placement Options
Three ways to surface the backlog on a phone, side by side. The desktop drawer lives on the right edge — none of these reuse that, so pick the native pattern that fits best.
A Bottom sheet + peeking pill My pick
A slim “Backlog” pill above the tab bar → taps open a half-height sheet
Strengths
  • Mirrors the desktop collapse → expand model, rotated to the thumb zone
  • Board (Todo row) stays visible behind the half sheet
  • Zero permanent board space; summoned only when needed
Trade-offs
  • Pulling uses a tap button, not drag (differs from desktop)
  • One extra element (the pill) sits above the tab bar

B Backlog as a 4th board row
A “Backlog” row below Done — drag a card up to Todo, same as any other row
Strengths
  • Consistent with the existing row pattern; nothing new to learn
  • Pull = drag up to Todo — exact parity with desktop
  • Always visible; backlog is never “hidden” behind a tap
Trade-offs
  • Permanently competes for vertical space with active columns
  • Backlog (staging) reads like a real status / column
  • A long backlog buries Done; lots of scrolling

C FAB + full-screen sheet
A floating button opens a full-screen backlog list — tap a card to pull it
Strengths
  • Maximum room — comfortable for a long backlog
  • Familiar FAB pattern; count badge is glanceable
  • Roomy rows can show full title + description
Trade-offs
  • Board is fully hidden while open — you lose the destination context
  • Drag-to-board impossible; pull is tap-only
  • FAB floats over board cards, can cover content
